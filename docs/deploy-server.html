Deploy signaling server

Pairing requires a small WebSocket server (free). GitHub Pages alone cannot relay the code.

One-time setup (~5 min)

  1. Click Deploy to Render below and connect your GitHub account
  2. Select repo flsourcing/Meta-Display-View-Caster — Render reads render.yaml automatically
  3. Wait until the service shows Live (first deploy ~3 min)
  4. Copy your service URL, e.g. https://meta-display-view-caster.onrender.com
  5. In GitHub, edit docs/config.js → set SIGNALING_URL: 'wss://YOUR-URL' (use wss://, not https://)
  6. Push to main — GitHub Pages updates in ~1 min
Deploy to Render (free)

Why? iPhone ↔ desktop pairing over PeerJS alone is unreliable (NAT / iOS Web App limits). The signaling server only exchanges the 6-digit code — video still goes phone → desktop via WebRTC.
